Documentación de Lógica de Datos — SENA APE 2026

1. Monitorización de Cola

Estas consultas alimentan el Dashboard del Coordinador para mostrar el estado actual de la oficina.

Conteo de Turnos Pendientes por Tipo

SELECT tur_tipo, COUNT(*) as cantidad
FROM turno
WHERE DATE(tur_hora_fecha) = CURDATE()
AND NOT EXISTS (
  SELECT 1 FROM atencion WHERE turno.tur_id = atencion.TURNO_tur_id
)
GROUP BY tur_tipo;
Lógica: Filtra solo los turnos generados "Hoy" que no tienen un registro asociado en la tabla `atencion`. Se agrupa por tipo para diferenciar la demanda entre categorías.

2. Visualización Pública (Pantalla)

Lógica aplicada para el ordenamiento legal y de servicio en la pantalla de turnos principal.

Ordenamiento por Prioridad SENA

SELECT * FROM turno
WHERE DATE(tur_hora_fecha) = CURDATE()
AND tur_id NOT IN (SELECT TURNO_tur_id FROM atencion)
ORDER BY
  CASE
    WHEN tur_tipo = 'Victimas' THEN 1
    WHEN tur_tipo = 'Prioritario' THEN 2
    ELSE 3
  END ASC,
  tur_id ASC;

3. Lógica de Asignación (Asesores)

Consultas críticas que incluyen integridad de datos y concurrencia.

Llamado de Siguiente Turno (Pessimistic Locking)

START TRANSACTION;
SELECT * FROM turno
WHERE tur_tipo = 'Victimas'
AND NOT EXISTS (SELECT 1 FROM atencion WHERE turno.tur_id = atencion.TURNO_tur_id)
ORDER BY tur_id ASC
FOR UPDATE;
INSERT INTO atencion (atnc_hora_inicio, ASESOR_ase_id, TURNO_tur_id, atnc_tipo) ...
COMMIT;
Seguridad: El uso de FOR UPDATE bloquea la fila del turno seleccionado hasta que el Asesor termina la transacción, evitando que dos asesores atiendan al mismo ciudadano simultáneamente.
