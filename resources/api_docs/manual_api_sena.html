Proyecto DigiTurno - Módulo APE 2026

1. Introducción

Este informe detalla los endpoints asíncronos desarrollados para la modernización del sistema de turnos. Estas APIs permiten la comunicación en tiempo real entre el servidor Laravel y las interfaces de usuario (Pantallas y Dashboards) sin necesidad de recarga manual.

2. Especificaciones de Endpoints

GET /api/pantalla/data

Propósito: Suministrar datos actualizados a la pantalla pública de turnos.

CampoDescripción
turnoActualInformación del turno que está siendo llamado en este momento.
turnosEnEsperaColección de turnos pendientes, ordenados por prioridad legal.
timestampSello de tiempo del servidor para control de latencia.

Ejemplo de Respuesta:

{
  "success": true,
  "turnoActual": { "tur_numero": "A-001", "modulo": 1 },
  "turnosEnEspera": [ { "tur_numero": "B-002", "tur_tipo": "Prioritario" } ]
}
GET /api/coordinador/stats

Propósito: Alimentar el tablero de monitoreo del Coordinador de Agencia.

CategoríaDescripción
GeneralTurnistas sin condiciones especiales.
PrioritarioAdultos mayores, personas con discapacidad, etc.
VictimasPoblación vulnerable con máxima prioridad de atención.

Frecuencia de Consumo: 300,000 ms (5 minutos).
